Types of XSS

1.Persistent XSS

- the malicious string originates from the website's database

2.Reflected XSS

- the malicious string originates from the victim's request

3.DOM-based XSS

- the vulnerability is in the client-side code rather than the server-side code

DOM-based XSS

What makes DOM-based XSS different?

In DOM-based XSS, the malicious JavaScript is executed at some point after the page has loaded.

XSS vulnerability appears in your website's client-side JavaScript code. Server side isn't affected!

Preventing XSS

1.Securing user input

2.Using Content Security Policy (CSP)

Securing user input

1.Encoding

which escapes the user input so that the browser interprets it only as data, not as code.

2.Validation

which filters the user input so that the browser interprets it as code without malicious commands.

Encoding


								<script>
									element.textContent = untrustedData; 
								</script>

Validation

Allowing some HTML elements (such as <em> and <strong>) but disallowing others (such as <script>)

Content Security Policy (CSP)

1.No untrusted sources

2.No inline resources

3.No eval

Enabling CSP on your website

Add additional HTTP header to your pages: Content‑Security‑Policy.


								<meta http-equiv="Content-Security-Policy" content="default-src https:">
